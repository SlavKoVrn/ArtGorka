# Project Management API

REST API для управления проектами на чистом PHP 8.0+.

## Требования
- PHP 8.0+
- MySQL 5.7+ или MariaDB
- Composer (опционально, для автозагрузки)

## Установка

1. Клонируйте репозиторий.
2. Настройте подключение к БД в файле `config/database.php`.
3. Импортируйте схему базы данных:
   ```bash
   mysql -u root -p < database/schema.sql
   ```
4. Настройте веб-сервер (Apache/Nginx) так, чтобы корневая директория (Document Root) указывала на папку `public/`.

## API Endpoints

### Получить список проектов
`GET /api/projects`
- Параметры фильтрации: `?status=production`

### Получить проект по ID
`GET /api/projects/{id}`

### Создать проект
`POST /api/projects`
```json
{
  "name": "My Website",
  "url": "https://example.com",
  "platform": "WordPress",
  "status": "development",
  "description": "Test project"
}
```

### Обновить проект
`PUT /api/projects/{id}`
(Передаются только поля, которые нужно обновить)

### Удалить проект
`DELETE /api/projects/{id}`

## Структура ответа
Все ответы возвращаются в формате JSON:
```json
{
  "success": true,
  "data": { ... },
  "message": "..."
}
```

## Безопасность
- Используется PDO с подготовленными выражениями (защита от SQL Injection).
- Валидация входных данных (URL, Enums).
- Конфигурация вынесена из кода.
```

### Ключевые особенности реализации:

1.  **Чистота кода:** Использованы пространства имен (`App\Controllers` и т.д.), что делает код модульным.
2.  **Безопасность:**
    *   `PDO::ATTR_EMULATE_PREPARES => false` гарантирует, что подготовка запросов происходит на стороне сервера БД, а не эмулируется PHP (важно для безопасности).
    *   Валидация URL через `filter_var`.
    *   Whitelist для `platform` и `status`.
3.  **Архитектура:**
    *   **Controller:** Только принимает запрос и возвращает ответ.
    *   **Model:** Только логика работы с данными.
    *   **Validator:** Отдельный класс для правил.
4.  **RESTful:** Правильные HTTP коды (201 для создания, 404 для не найдено, 400 для ошибок валидации).
5.  **Гибкость:** Динамический `UPDATE` запрос в модели позволяет обновлять только переданные поля, не перезаписывая остальные NULL-значениями.
